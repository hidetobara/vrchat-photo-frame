Sync Frame ver0.2

# 概要
Sync Frame は、Google スプレッドシートで画像URLを管理することで、
ワールドの更新することなく、ポスターなどの画像を更新することができます。

# 使い方
## Google Spread Sheet 上
ワークシート1つ(スプレッドシート下部のタブ)を作り、名前をアルファベットで付けます。
1列目にID(アルファベットと数字、アンダーバー)を、2列目に画像への URL を入力してください。
3列目以降は参照しないので、コメントなどを記述して問題ありません。

共有ボタンを押し、「リンクを知っている人全員」「編集者」にしてください。
そのリンクをメモします。

## Google Drive 内の画像
Google ドライブ上で、対象となる画像に対し、共有ボタンを押し、「リンクを知っている人全員」「閲覧者」にして、URLを取得します。
その URL を、他の画像の時と同様に、ワークシートの2列目に入力します。

## Twitter の画像
以下のようなURLを、ワークシートの2列目に入力します。
https://twitter.com/hidetobara/status/1752354104247189536/photo/1

## ドラッグ＆ドロップ
2列目の URL を空にしていると、後述のブラウザ・ツールから
画像をドラッグ＆ドロップで更新するできるようになります。

# Unity 上
Sync Frame の Unity packaege をインポートします。

Sync　Manager の prefab をワールド内に設置します。
Sync Manager のインスペクター内の順番に従い、
まず、スプレッドシートの URL を入力します。
Work Sheet は、ワークシート名(下のタブ名)を入力します。

読み込みボタンによって、このスプレッドシートが読み込まれます。
アップロードボタンによって、スプレッドシート内に指定された画像をウェブ上へと同期します。

新しく表示用の Frame を追加することもできます。
その際、スプレッドシートの1列目の ID を入力することで、
その画像リンクと同期させることができます。

そして、ボタン「仮表示」を押してください。

# より進んだ設定
- Sync Frame のインスペクター
  - Auto Adjust Aspect は読み込む画像のアスペクト比に応じて、フレーム縦横比を調整します。
    - Transform の Scale の最大値をもとにフレームの大きさは決まります。
  - Frame Width を 0 にすると、フレームは無くなります。
  - Loading Type は、読み込み中のエフェクトです。現在、「ぐるぐる」と「モザイク」があります。

# 注意事項
VRChat アプリ側で画像のダウンロードは、5秒間に画像1枚の間隔でしか許可していません。
そのため、ワールド内にダウンロードすべき画像の枚数が増えると、表示されるまで非常に時間がかかるようになります。
表示ラグを減らすために、なるべく画像の枚数は減らしましょう。
